# 🎯 전문가 포트폴리오 웹페이지 빌더
 
강사, 전문가, 작가, 코치, 컨설턴트, 프리랜서 등 **개인 전문가**를 위한 포트폴리오 웹페이지를 체계적으로 기획·제작하는 스킬입니다.
 
## 핵심 철학
 
전문가 포트폴리오는 단순한 이력서가 아니라 **"이 사람에게 배우고 싶다 / 함께 일하고 싶다"**는 감정을 불러일으키는 브랜딩 도구입니다. 
따라서 세 가지를 동시에 달성해야 합니다:
1. **신뢰 구축** — 경력, 실적, 자격증으로 전문성 증명
2. **매력 전달** — 차별화된 스토리와 비주얼로 기억에 남는 인상
3. **행동 유도** — 수강 신청, 상담 예약, 협업 문의 등 전환 달성

## 📋 5단계 워크플로우
 
### STEP 1: 전문가 프로필 인터뷰
 
사용자에게 다음 정보를 수집합니다. 사용자가 이미 제공한 정보는 건너뛰세요.
 
#### 필수 정보
| 항목 | 질문 예시 |
|------|----------|
| **전문가 유형** | 강사 / 컨설턴트 / 작가 / 코치 / 프리랜서 / 기타 |
| **이름 & 직함** | 표시될 이름과 대표 타이틀 |
| **핵심 전문 분야** | 무엇을 가르치거나 어떤 서비스를 제공하는지 (1-3줄) |
| **대표 슬로건** | 한 줄로 자신을 표현하는 문구 (없으면 함께 만들기) |
| **주요 경력** | 핵심 이력 3-7개 (시간 순 또는 중요도 순) |
| **대표 실적/성과** | 수치화된 성과 (수강생 수, 저서 수, 프로젝트 수 등) |
| **연락처/CTA** | 이메일, 전화, 카카오톡, 상담 예약 링크 등 |
 
#### 선택 정보 (있으면 품질 향상)
| 항목 | 설명 |
|------|------|
| **저서/출판물** | 책, 논문, 칼럼 등 |
| **수상/인증** | 자격증, 수상 경력, 인증 |
| **강연/미디어** | 강연 이력, 미디어 출연, 팟캐스트 등 |
| **고객 후기** | 추천사, 수강 후기 (실명/익명) |
| **포트폴리오 작품** | 프로젝트 이미지, 결과물 스크린샷 |
| **SNS/외부 링크** | YouTube, Instagram, LinkedIn, 블로그 등 |
| **프로필 사진** | 제공 시 배치, 미제공 시 플레이스홀더 |
| **선호 색상/분위기** | 원하는 디자인 무드 |
 
인터뷰 시 한 번에 너무 많은 질문을 하지 말고 **3-5개씩 나눠서** 자연스럽게 대화하며 수집합니다.
정보가 부족해도 합리적인 기본값과 플레이스홀더로 먼저 초안을 생성한 뒤 사용자와 반복 수정합니다.

### STEP 2: 포트폴리오 구조 설계
 
전문가 유형에 따라 최적화된 섹션 구조를 선택합니다.
 
#### 전문가 유형별 섹션 구성
 
**🎓 강사/교육자 유형**
```
1. Hero (이름, 타이틀, 슬로건, CTA)
2. About (소개 스토리 + 교육 철학)
3. Expertise (전문 분야 카드)
4. Achievements (숫자로 보는 성과: 수강생 수, 강의 시간, 만족도)
5. Curriculum (대표 강의/커리큘럼 소개)
6. Publications (저서/출판물)
7. Testimonials (수강생 후기)
8. Media (미디어 출연, YouTube 등)
9. Contact/CTA (수강 신청, 상담 예약)
```
 
**✍️ 작가/크리에이터 유형**
```
1. Hero (이름, 대표작 소개, 슬로건)
2. About (작가 소개 + 창작 철학)
3. Works (대표 작품 갤러리/리스트)
4. Awards (수상 이력)
5. Press (미디어 리뷰, 인터뷰)
6. Events (출간 기념회, 북토크, 전시)
7. Testimonials (독자/평론가 후기)
8. Contact/CTA (출간 문의, 강연 요청)
```
 
**💼 컨설턴트/코치 유형**
```
1. Hero (이름, 전문 분야, 핵심 가치 제안)
2. About (경력 스토리 + 접근 방법론)
3. Services (서비스 패키지 소개)
4. Process (컨설팅/코칭 프로세스 시각화)
5. Results (클라이언트 성과 사례 — Before/After)
6. Credentials (자격, 인증, 학력)
7. Testimonials (클라이언트 추천사)
8. FAQ (자주 묻는 질문)
9. Contact/CTA (무료 상담 신청)
```
 
**🛠️ 프리랜서/크리에이티브 유형**
```
1. Hero (이름, 직무, 포트폴리오 하이라이트)
2. About (자기소개 + 작업 스타일)
3. Skills (기술 스택/역량 시각화)
4. Portfolio (프로젝트 갤러리 — 썸네일+설명)
5. Experience (주요 경력 타임라인)
6. Testimonials (클라이언트 후기)
7. Contact/CTA (프로젝트 문의)
```
 
공통 원칙:
- Hero 섹션은 반드시 **3초 안에** 이 전문가가 누구이고 무엇을 하는지 파악 가능해야 함
- Achievements/Numbers 섹션은 가능하면 포함 — 숫자는 신뢰를 빠르게 구축
- CTA는 최소 2곳 (Hero + Footer) 배치
- 모바일 퍼스트 반응형 설계

### STEP 3: 디자인 시스템 결정
 
사용자의 선호나 전문 분야에 맞는 디자인 무드를 제안합니다.
 
#### 프리셋 디자인 무드 (5종)
 
**1. 📘 Authority (권위형)** — 교수, 의사, 법률가, 금융 전문가
- 색상: 네이비(#1B2A4A) + 골드(#C9A84C) + 화이트
- 폰트: 서체가 주는 무게감 (Noto Serif KR + Pretendard)
- 스타일: 클래식하고 절제된, 여백 넉넉한 레이아웃
- 배경: 순백 또는 옅은 크림, 미세한 종이 텍스처
 
**2. 🎨 Creative (크리에이티브형)** — 디자이너, 작가, 아티스트, 크리에이터
- 색상: 다크 차콜(#2D2D2D) + 비비드 포인트(#FF6B35 또는 #E63946)
- 폰트: 개성있는 디스플레이 + 깔끔한 본문 (Playfair Display + Pretendard)
- 스타일: 비대칭 레이아웃, 대담한 타이포그래피, 이미지 중심
- 배경: 다크모드 기본, 그래디언트 오버레이
 
**3. 🌿 Warm & Approachable (친근형)** — 코치, 상담사, 교육자 (중장년 대상)
- 색상: 딥그린(#2D5F3E) + 웜베이지(#F5F0E8) + 소프트 코랄(#E8927C)
- 폰트: 둥글고 따뜻한 느낌 (Noto Sans KR + Gowun Dodum)
- 스타일: 부드러운 곡선, 충분한 여백, 큰 폰트
- 배경: 웜톤 베이지/크림, 부드러운 그래디언트
 
**4. ⚡ Tech & Modern (테크형)** — IT 강사, 개발자, 데이터 전문가
- 색상: 다크(#0F0F0F) + 사이버 블루(#00D4FF) + 네온 그린(#00FF88)
- 폰트: 모노스페이스 + 기하학적 산세리프 (JetBrains Mono + Pretendard)
- 스타일: 코드 에디터 감성, 그리드 패턴, 터미널 스타일 요소
- 배경: 순수 다크, 미세한 도트 그리드/그래디언트 메시
 
**5. 🌸 Elegant & Refined (우아형)** — 라이프스타일 전문가, 뷰티, 웰니스, 문학
- 색상: 소프트핑크(#F2E2D2) + 딥와인(#6B2D3E) + 골드(#B8860B)
- 폰트: 섬세한 세리프 + 모던 산세리프 (Cormorant Garamond + Pretendard)
- 스타일: 에디토리얼 레이아웃, 섬세한 보더 라인, 우아한 여백
- 배경: 소프트 뉴트럴, 미세한 패턴 오버레이
 
사용자가 특정 색상이나 분위기를 요청하면 프리셋을 기반으로 커스터마이즈합니다.
사용자가 선호를 모르면, 전문가 유형을 기반으로 가장 적합한 무드를 추천합니다.

### STEP 4: 코드 생성
 
#### 기술 스택
- **기본**: HTML + CSS + JavaScript (단일 파일, 즉시 배포 가능)
- **폰트**: Google Fonts (한글: Pretendard, Noto Sans KR, Gowun Dodum / 영문: 무드별 선택)
- **아이콘**: Lucide Icons CDN 또는 SVG 인라인
- **애니메이션**: CSS Animations + Intersection Observer (스크롤 애니메이션)
- **반응형**: 모바일 퍼스트 (breakpoints: 768px, 1024px, 1280px)
 
#### 코드 품질 기준
 
1. **시맨틱 HTML**: section, article, header, footer 등 의미론적 태그 사용
2. **CSS 변수**: 색상, 폰트, 간격 등을 CSS 커스텀 프로퍼티로 관리
3. **반응형 완벽 구현**: 모바일 → 태블릿 → 데스크탑 모두 레이아웃 최적화
4. **접근성**: alt 텍스트, aria-label, 키보드 네비게이션, 충분한 색상 대비
5. **성능**: 이미지 lazy loading, 최소한의 JS, CSS 애니메이션 우선
6. **SEO 기본**: meta 태그, Open Graph, 구조화된 마크업
 
#### 섹션별 구현 패턴
 
**Hero 섹션**
```
- 풀스크린 또는 대형 Hero 영역
- 프로필 이미지 (원형 또는 마스크 + 그림자/보더 효과)
- 이름 (H1, 큰 사이즈) + 직함/타이틀 (서브헤딩)
- 슬로건 또는 한 줄 소개
- CTA 버튼 1-2개 (주 액션 + 보조 액션)
- 배경: 그래디언트, 패턴, 또는 미묘한 애니메이션
- 스크롤 다운 인디케이터 (선택)
```
 
**About 섹션**
```
- 2컬럼 레이아웃 (사진 + 텍스트) 또는 풀-텍스트
- 스토리텔링 형식의 자기소개 (3-5 문단)
- 교육/작업 철학 강조
- 핵심 키워드 하이라이트
```
 
**Achievements/Numbers 섹션**
```
- 카운터 애니메이션 (IntersectionObserver + CSS counter 또는 JS)
- 3-4개의 핵심 수치 (큰 숫자 + 라벨)
- 아이콘 또는 일러스트 동반
- 예: "2,500+ 수강생 | 15년 경력 | 12권 저서 | 98% 만족도"
```
 
**Expertise/Services 섹션**
```
- 카드 그리드 (2-3열)
- 각 카드: 아이콘 + 제목 + 설명 (2-3줄)
- 호버 효과 (색상 변경, 그림자, 스케일)
```
 
**Portfolio/Works 섹션**
```
- 갤러리 그리드 또는 캐러셀
- 썸네일 이미지 + 제목 + 카테고리 태그
- 이미지 없는 경우: 플레이스홀더 + 텍스트 카드
- 저서인 경우: 책 표지 목업 스타일
```
 
**Timeline/Experience 섹션**
```
- 수직 타임라인 (왼쪽 또는 교차 배치)
- 각 항목: 연도 + 제목 + 설명
- 스크롤 시 순차 등장 애니메이션
```
 
**Testimonials 섹션**
```
- 카드 캐러셀 또는 그리드
- 인용문 스타일 (큰 따옴표 아이콘)
- 후기 작성자 이름 + 소속 (가능하면)
- 별점 또는 평점 표시 (선택)
```
 
**Contact/CTA 섹션**
```
- 강렬한 배경색 또는 그래디언트
- 명확한 행동 유도 메시지
- CTA 버튼 (큰 사이즈, 눈에 띄는 색상)
- 연락처 정보 (이메일, 전화, SNS 아이콘 링크)
- 간단한 문의 폼 (이름, 이메일, 메시지) — 선택
```
 
**Footer**
```
- 저작권 표시
- SNS 아이콘 링크
- 네비게이션 링크 (앵커)
- "Powered by..." 또는 브랜드 로고 (선택)
```
 
#### 필수 인터랙션
 
1. **스무스 스크롤**: 네비게이션 앵커 링크 클릭 시 부드러운 스크롤
2. **스크롤 애니메이션**: 섹션 진입 시 fade-in / slide-up 효과 (IntersectionObserver)
3. **네비게이션**: 고정(sticky) 헤더 + 현재 섹션 하이라이트
4. **모바일 메뉴**: 햄버거 메뉴 + 슬라이드 인 네비게이션
5. **카운터 애니메이션**: 숫자 섹션에서 카운트업 효과
 
#### 이미지 처리

## ⚠️ 주의사항
 
### DO ✅
- 전문가의 **고유한 가치**를 전면에 배치
- 수치와 구체적 성과로 신뢰 구축
- 모바일 환경에서도 완벽하게 작동
- CSS 변수를 활용해 쉽게 커스터마이즈 가능한 구조
- 시맨틱 HTML로 SEO 기본 최적화
- 실제 전문가가 사용하는 톤과 용어 반영
 
### DON'T ❌
- 과도한 애니메이션으로 로딩 지연 유발
- 텍스트 없이 이미지만으로 핵심 정보 전달
- 5개 이상의 CTA 버튼 배치
- 자동 재생 동영상/음악
- 너무 작은 폰트 (본문 16px 미만)
- 대비가 낮은 색상 조합 (접근성 위반)
- 허위/과장된 경력 작성 조장

## 🎨 frontend-design 스킬 연계
 
이 스킬은 코드 생성 시 **frontend-design 스킬의 디자인 철학**을 적극 반영합니다:
- **타이포그래피**: 무드에 맞는 독특하고 아름다운 폰트 선택 (Inter, Arial 같은 범용 폰트 지양)
- **색상**: CSS 변수로 일관된 테마, 대담한 메인 컬러 + 날카로운 액센트
- **모션**: 스크롤 트리거 애니메이션, 호버 효과, 스태거드 등장 효과
- **공간 구성**: 의도적인 비대칭, 충분한 여백, 시각적 위계 질서
- **배경 & 디테일**: 솔리드 컬러 기본값 탈피 — 그래디언트, 텍스처, 패턴, 오버레이 적극 활용
 
**제네릭한 "AI 느낌" 디자인을 철저히 배제하고, 각 전문가의 개성을 살리는 고유한 디자인을 만듭니다.**
